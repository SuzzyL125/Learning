------------------------------------------
Basic Unix Commands
------------------------------------------

The Unix shell interprets commands like date, echo, and pwd (print working directory) to interact with the system.
Commands not recognized by the shell return an error, indicating the command is not found.

File System Navigation

The Unix file system is hierarchical, starting from the root directory, with directories and subdirectories separated by slashes (/).
The cd command changes directories using either absolute paths (from root) or relative paths (from the current directory).
Special directory references include "." for the current directory and ".." for the parent directory.

Listing and Viewing Files

The ls command lists files in the current directory, showing files alphabetically by default.
Additional options like ls -l provide detailed file information including type, permissions, owner, size, and modification date.
The ls -lt option lists files in reverse chronological order based on modification time.
The man command (e.g., man ls) displays the manual pages for commands to learn more about their options and usage.

------------------------------------------
Naming Patterns for File Management
------------------------------------------

Files for different species and types can be numerous, making manual listing inefficient.
Using naming patterns helps manage and list files effectively.

Wildcards in Unix

The asterisk () wildcard represents any string of characters, useful for listing all files related to a species (e.g., apple.).
The question mark (?) wildcard represents a single character, allowing more specific matches (e.g., ?pple.genome).

Character Ranges and Combinations

Square brackets [] specify a range of characters, useful for matching files with certain patterns (e.g., [a-z]*.genome).
Combining wildcards can refine searches, such as listing genome files starting with a specific letter (e.g., p*.genome).

Curly Brackets for Multiple Options

Curly brackets {} allow specifying multiple options in one command (e.g., {pear,peach}.genome lists files starting with pear or peach).
This method efficiently handles large sets of files sharing common naming patterns.

------------------------------------------
Creating and Organizing Directories
------------------------------------------

Use the mkdir command to create directories for different species (e.g., apple, pear, peach).
Organize files by copying or moving species-specific files into their respective directories.

Copying and Moving Files

Copy files with the cp command, either individually or using wildcards (e.g., apple.*).
Move files with the mv command to transfer them from the current directory to target directories, effectively removing them from the original location.

Removing Files and Directories

Remove files with the rm command; use the -i option for interactive confirmation to avoid accidental deletion.
Remove empty directories with rmdir; for non-empty directories, use rm -r to recursively delete all contents and the directory itself.
!The rmdir command only removes empty directories.
------------------------------------------
Unix Commands for Accessing Files
------------------------------------------

The "more" command allows viewing file content one page or line at a time, useful for large files.
The "less" command provides similar functionality but also allows backward navigation through the file.

------------------------------------------
Commands for Viewing File Content
------------------------------------------

The command head displays the top lines of a file, defaulting to 10 lines, but can be adjusted with options like -50 to show the first 50 lines.
The command tail shows the last lines of a file, also defaulting to 10 lines, with options such as -15 to display the last 15 lines.

Using cat for File Content

The cat command (short for concatenate) can display the entire content of a file or combine multiple files into one continuous output.
When used on large files, cat outputs everything at once, which may require capturing the output for easier viewing later.

File Examples in Genomic Data

Example files include peach.genome, peach.genes, and peach.samples, containing genomic sequences and gene variant information.
These commands help explore specific parts of large genomic data files efficiently without loading the entire file at once.

------------------------------------------
Standard Input and Output in Unix
------------------------------------------

Unix commands like cat and wc read input from files or the command line and display output to the terminal.
There are three standard files: standard input, standard output, and standard error, which handle data flow in Unix.

Redirecting Input and Output

The greater than sign (>) redirects command output from the terminal to a file.


Sorting Content

The sort command arranges file lines alphabetically by default, but can be customized to sort by specific columns and numerically.
Sorting can be done in reverse order and by multiple columns, such as sorting months by season and then by their numerical order within the year.

Extracting and Manipulating Fields

The cut command extracts specific columns or ranges of columns from a file, with the ability to specify delimiters other than tabs.
Modifying delimiters in files affects how cut interprets columns, allowing flexible extraction of data fields.

Finding Unique Entries and Counting Occurrences

The uniq command filters out consecutive duplicate lines and can count occurrences of repeated lines.
Combining uniq with sort -u helps identify unique entries regardless of their order in the file.

Searching for Patterns

The grep command searches for specific patterns within files, showing matching lines and optionally their line numbers.
Grep supports searching with quoted strings and can be used to find specific data points, such as samples from a particular plant part.

Compression Tools

gzip compresses single files using the Lempel-Ziv algorithm, creating .gz files; gunzip decompresses them without data loss.
bzip2 offers stronger compression using the Burrows-Wheeler algorithm, producing .bz2 files; bunzip2 decompresses these files.

Archiving Multiple Files and Directories

tar creates archives by combining multiple files or entire directories into a single .tar file without compression.
Options like -c (create), -v (verbose), and -f (file) control tar's behavior.
After archiving, gzip or bzip2 can compress the .tar archive to reduce size further.

Extracting and Viewing Archives

To extract files, decompress the archive (gunzip or bunzip2) and then use tar with -x (extract), -v, and -f options.
The zcat command allows viewing the contents of compressed files without decompressing them first.
